# LiteLLM Proxy 操作手册

通过 LiteLLM Proxy 将 Claude Code 的请求转发到 DashScope 的 qwen3.7-plus 模型。

## 架构

```
Claude Code (VS Code)
    │  Anthropic Messages API
    │  model: claude-sonnet-4-6 / claude-opus-4-8 / ...
    │  Base URL: http://localhost:4000
    ▼
LiteLLM Proxy (localhost:4000)
    │  Anthropic → OpenAI 协议转换
    │  drop_params: true
    ▼
DashScope API (dashscope.aliyuncs.com)
    │  OpenAI 兼容模式
    ▼
qwen3.7-plus
```

## 项目文件说明

| 文件 | 说明 |
|------|------|
| `.env` | DashScope API Key 和 Base URL（敏感信息，已 gitignore） |
| `config.yaml` | LiteLLM 代理配置，映射 Claude 模型名到 qwen3.7-plus |
| `start.ps1` | PowerShell 启动脚本，自动加载 `.env` 并启动代理 |
| `start.bat` | CMD 启动脚本，功能同 `start.ps1` |
| `requirements.txt` | Python 依赖清单 |
| `test_litellm_sdk.py` | SDK 直接调用测试脚本 |
| `.claude/settings.local.json` | Claude Code 项目级环境变量配置（Base URL 指向本地代理） |

## 前置条件

- Python 3.8+
- Conda 环境（推荐）
- DashScope API Key（[申请地址](https://dashscope.console.aliyun.com/)）

## 安装步骤

### 1. 创建 Conda 环境

```powershell
conda create -n litellm python=3.11 -y
conda activate litellm
```

### 2. 安装依赖

```powershell
pip install -r requirements.txt
```

或手动安装核心依赖：

```powershell
pip install litellm[proxy] python-dotenv
```

验证安装：

```powershell
litellm --version
```

### 3. 配置环境变量

确保 `.env` 文件包含：

```
DASHSCOPE_API_KEY="你的API密钥"
DASHSCOPE_BASE_URL=https://dashscope.aliyuncs.com/compatible-mode/v1
```

> **重要**：`DASHSCOPE_BASE_URL` 必须设置，否则请求会被错误路由到 `api.openai.com` 导致连接超时。

### 4. 检查配置文件

`config.yaml` 已配置好模型映射，关键配置项：

```yaml
model_list:
  - model_name: qwen3.7-plus
    litellm_params:
      model: openai/qwen3.7-plus
      api_key: os.environ/DASHSCOPE_API_KEY
      api_base: os.environ/DASHSCOPE_BASE_URL

  - model_name: deepseek-v4-pro
    litellm_params:
      model: openai/deepseek-v4-pro
      api_key: os.environ/DEEPSEEK_API_KEY
      api_base: os.environ/DEEPSEEK_BASE_URL

  - model_name: claude-opus-4-8
    litellm_params:
      model: openai/qwen3.7-plus
      api_key: os.environ/DASHSCOPE_API_KEY
      api_base: os.environ/DASHSCOPE_BASE_URL

  - model_name: claude-sonnet-4-6
    litellm_params:
      model: openai/qwen3.7-plus
      api_key: os.environ/DASHSCOPE_API_KEY
      api_base: os.environ/DASHSCOPE_BASE_URL

litellm_settings:
  drop_params: true
  set_verbose: true
  default_model: openai/qwen3.7-plus

general_settings: {}
```

> **重要**：必须在 `model_list` 中为每个 Claude 模型名添加映射条目，同时在 `litellm_settings` 中设置 `default_model` 兜底。否则 Claude Code 请求中携带的 `claude-opus-4-8` 等模型名会因找不到对应模型而返回 400 错误。

> **注意**：`config.yaml` 中**不要使用中文注释**，Windows GBK 编码会导致 YAML 解析错误。

## 启动代理

### 方式一：PowerShell 脚本（推荐）

```powershell
conda activate litellm
cd E:\AI\projects\litellm_demo
.\start.ps1
```

#### 为什么需要 `start.ps1`？

**核心问题**：LiteLLM 不会自动加载 `.env` 文件。

`config.yaml` 中使用 `os.environ/DASHSCOPE_API_KEY` 和 `os.environ/DASHSCOPE_BASE_URL` 引用环境变量。如果直接运行 `litellm --config config.yaml`，这些环境变量为空，LiteLLM 会将请求路由到默认的 `api.openai.com`，导致连接超时。

**解决方案**：`start.ps1` 在启动 LiteLLM 之前，手动解析 `.env` 文件并注入环境变量。

#### `start.ps1` 完整内容

```powershell
# Load .env file and collect keys
$keys = @()
Get-Content .env | ForEach-Object {
    if ($_ -match '^\s*([^#][^=]+)=(.*)$') {
        $key = $Matches[1].Trim()
        $val = $Matches[2].Trim().Trim('"').Trim("'")
        [System.Environment]::SetEnvironmentVariable($key, $val, "Process")
        $keys += $key
    }
}

# Display all loaded env vars
foreach ($key in $keys) {
    $val = [System.Environment]::GetEnvironmentVariable($key, "Process")
    if ($val.Length -gt 10) {
        Write-Host "$key = $($val.Substring(0,10))..." -ForegroundColor Green
    } else {
        Write-Host "$key = $val" -ForegroundColor Green
    }
}
Write-Host "Starting LiteLLM Proxy..." -ForegroundColor Cyan

litellm --config config.yaml --port 4000
```

#### 逐行解释

| 行号 | 代码 | 作用 |
|------|------|------|
| 1 | `$keys = @()` | 初始化数组，记录已加载的变量名 |
| 2 | `Get-Content .env` | 读取 `.env` 文件的所有行 |
| 3 | `ForEach-Object { ... }` | 逐行处理 |
| 4 | `if ($_ -match '^\s*([^#][^=]+)=(.*)$')` | 正则匹配：跳过注释行（`#` 开头）和空行，提取 `KEY=VALUE` |
| 5 | `$key = $Matches[1].Trim()` | 提取键名，去除两端空格 |
| 6 | `$val = $Matches[2].Trim().Trim('"').Trim("'")` | 提取值，去除两端空格和引号 |
| 7 | `[System.Environment]::SetEnvironmentVariable(...)` | 将变量注入当前进程的环境变量 |
| 8 | `$keys += $key` | 记录变量名 |
| 10-17 | `foreach ($key in $keys) { ... }` | 遍历并打印所有已加载的变量（长值只显示前 10 位） |
| 18 | `litellm --config config.yaml --port 4000` | 启动 LiteLLM 代理 |

> **注意**：PowerShell 执行策略可能阻止脚本运行。如遇此问题，先执行：
> ```powershell
> Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
> ```

## 配置 Claude Code

项目已配置 `.claude/settings.local.json`，Claude Code 会自动加载：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:4000",
    "ANTHROPIC_API_KEY": "any-value"
  }
}
```

或手动设置环境变量：

```powershell
$env:ANTHROPIC_BASE_URL = "http://localhost:4000"
$env:ANTHROPIC_API_KEY = "any-value"
```

> **说明**：当前配置未启用 `master_key`（无数据库模式），`ANTHROPIC_API_KEY` 可以是任意值。如需启用认证，需同时配置数据库。

## 后台运行

直接启动 LiteLLM 会占用终端窗口，关闭窗口即停止服务。以下是几种后台运行方案。

### 方案 A：NSSM 注册为 Windows 服务（推荐）

[NSSM](https://nssm.cc/) 可将任意程序注册为 Windows 服务，支持开机自启、崩溃自动重启。

1. 下载 NSSM：https://nssm.cc/download
2. 安装服务：`nssm install LiteLLMProxy`
3. 配置路径、参数、启动目录
4. 管理服务：`nssm start/stop/restart LiteLLMProxy`

### 方案 B：PowerShell 后台任务

```powershell
Start-Process -WindowStyle Hidden -FilePath "powershell" -ArgumentList "-ExecutionPolicy Bypass -File .\start.ps1"
```

### 方案 C：Docker

```powershell
docker run -d --name litellm -p 4000:4000 --restart always litellm-proxy
```

### 方案 D：Linux systemd

创建 `/etc/systemd/system/litellm.service`，使用 `systemctl enable/start litellm`。

### 方案 E：tmux / screen

```bash
tmux new -s litellm
litellm --config config.yaml --port 4000
# Ctrl+B, D 分离会话
```

详细配置见 [STARTUP.md](STARTUP.md)。

## 在新项目中使用

### 方案一：共用代理（推荐）

LiteLLM 代理只需启动一次，所有项目共享。新项目只需配置 Claude Code：

1. 确保 LiteLLM 代理已在运行
2. 在新项目根目录创建 `.claude/settings.local.json`：

```json
{
  "env": {
    "ANTHROPIC_BASE_URL": "http://localhost:4000",
    "ANTHROPIC_API_KEY": "any-value"
  }
}
```

3. 确保 `.claude/settings.local.json` 已 gitignore

### 方案二：独立部署

复制 `config.yaml`、`.env`、`start.ps1`/`start.bat` 到新项目，修改启动端口避免冲突。

## 常见问题

### 错误：Cannot connect to Host api.openai.com:443

**原因**：`DASHSCOPE_BASE_URL` 环境变量未设置，LiteLLM 将请求发到了默认的 OpenAI 地址。

**解决**：使用 `start.ps1` 启动，或手动设置 `$env:DASHSCOPE_BASE_URL` 后重启 LiteLLM。

### 错误：UnicodeDecodeError / YAML parse error

**原因**：`config.yaml` 中包含中文字符，Windows 默认 GBK 编码导致 YAML 解析失败。

**解决**：`config.yaml` 中只使用英文注释，确保文件保存为 UTF-8 无 BOM 编码。

### 错误：401 Unauthorized

**原因**：如果启用了 `master_key`，Claude Code 的 `ANTHROPIC_API_KEY` 必须与之匹配。

**解决**：当前配置未启用认证（`general_settings: {}`），不应出现此错误。如需启用认证，在 `config.yaml` 中添加 `master_key` 并确保 `.claude/settings.local.json` 中的 `ANTHROPIC_API_KEY` 与之匹配。

### 错误：PowerShell 无法加载脚本

**原因**：PowerShell 执行策略限制了脚本运行。

**解决**：

```powershell
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
```

### 错误：pip 依赖冲突（如 rich 版本冲突）

**原因**：环境中已有包与 litellm 依赖版本不兼容。

**解决**：一般不影响使用。如需彻底解决，使用独立的 conda 环境。

### 错误：No connected db

**原因**：配置了 `master_key` 但未配置数据库，LiteLLM 无法验证 API 密钥。

**解决**：二选一：
- 移除 `master_key`（无需认证）：将 `general_settings` 设为空 `{}`
- 添加数据库配置：`database_url: "sqlite:///:memory:"` 或 PostgreSQL 连接串

### 错误：Invalid model name passed in model=claude-xxx

**原因**：Claude Code 在 API 请求中会硬编码发送 Claude 模型名（如 `claude-opus-4-8`、`claude-sonnet-4-6`），不受 `/model` 命令影响。如果 `model_list` 中没有对应条目，LiteLLM 会返回 400 错误。

**解决**：在 `config.yaml` 的 `model_list` 中添加 Claude 模型名的映射，并在 `litellm_settings` 中设置 `default_model` 兜底：

```yaml
model_list:
  - model_name: claude-opus-4-8
    litellm_params:
      model: openai/qwen3.7-plus
      ...
  - model_name: claude-sonnet-4-6
    litellm_params:
      model: openai/qwen3.7-plus
      ...

litellm_settings:
  default_model: openai/qwen3.7-plus
```

> **提示**：如果后续 Claude Code 升级使用新的模型名，只需在 `model_list` 中添加对应条目即可。

## Claude Code Settings 加载优先级

1. `~/.claude/settings.json` — 用户全局配置
2. `<project>/.claude/settings.json` — 项目共享配置（可提交到 git）
3. `<project>/.claude/settings.local.json` — 项目本地配置（不提交到 git，优先级最高）

同名环境变量会被项目级覆盖，不会与全局配置冲突。

## 数据库（可选）

数据库仅在以下场景需要：
- 虚拟密钥管理（多用户/多团队）
- RPM/TPM 限制
- 使用统计和计费

如需启用，在 `config.yaml` 中添加：

```yaml
general_settings:
  database_url: "postgresql://user:password@host:port/dbname"
```

## 关键要点

- **协议转换**：LiteLLM 自动将 Anthropic Messages API 格式转换为 DashScope 的 OpenAI 兼容格式
- **模型映射**：Claude Code 无论选择哪个 Claude 模型名，都会被路由到 qwen3.7-plus
- **drop_params**：自动丢弃 DashScope 不支持的参数，避免请求报错
- **认证**：当前未启用 `master_key`，无需数据库；如需认证需同时配置数据库
- **Windows 编码**：`config.yaml` 中不要使用中文注释，否则 YAML 解析器在 Windows 上会因 GBK 编码报错
